# PRD - 讀書筆記本系統

## 1. 專案概述

- **背景與動機**：學生在閱讀課外讀物或教科書時，常缺乏有系統的工具來整理重點與心得。傳統紙本筆記不易搜尋，一般的備忘錄軟體則缺乏針對「書籍與閱讀」設計的專屬欄位。
- **目標用戶**：各學齡層之學生（小學生至研究生）。
- **核心價值主張**：提供一個簡單、直覺且專注的系統，讓學生能輕鬆記錄、分類並快速檢索每一本書的學習重點與心得評分，將知識妥善保存。

## 2. 功能需求

以下列出本系統的主要功能與對應的使用者故事：

1. **書籍筆記新增與管理**
   - 作為學生，我希望能夠新增書籍筆記（包含書名、心得內容、評分），以便完整記錄我的閱讀歷程與想法。
2. **筆記列表與瀏覽**
   - 作為學生，我希望能在系統首頁看到所有已建立的讀書筆記列表，以便快速瀏覽我讀過的所有書籍。
3. **關鍵字搜尋功能**
   - 作為學生，我希望能夠透過輸入關鍵字來搜尋書名、心得內容或評分，以便在需要尋找過去資料時能快速定位到特定的筆記。
4. **筆記分類與標籤系統**
   - 作為學生，我希望能夠為每一篇筆記加上分類或標籤（如：文學、歷史、專業科目），以便根據不同領域整理與篩選我的閱讀紀錄。
5. **筆記編輯與刪除**
   - 作為學生，我希望能夠隨時回頭修改心得內容或刪除不再需要的筆記，以便維持筆記內容的準確性與整潔。

## 3. 非功能需求

- **技術限制**：系統須採用 Flask 作為後端框架，前端使用 Jinja2 模板引擎進行渲染，並搭配 SQLite 作為輕量級關聯式資料庫。
- **效能考量**：頁面載入時間與搜尋回應時間應盡可能保持在 2 秒內，提供流暢的操作體驗。
- **安全考量**：系統需實作基本的防護機制，如防止 SQL Injection 與 XSS（跨站腳本攻擊），確保使用者填寫的心得內容能被安全地處理與呈現。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須具備)**：
  - 新增讀書筆記（欄位至少包含：書名、心得、評分）。
  - 檢視所有筆記的列表頁面。
  - 關鍵字搜尋功能（可搜尋書名與心得）。
- **Should Have (應該具備)**：
  - 編輯與刪除現有筆記的功能。
  - 基礎的表單驗證（例如書名不能為空）。
- **Nice to Have (有會更好)**：
  - 自訂標籤或分類功能。
  - 簡單的閱讀統計數據（例如：總閱讀本數、平均評分）。
